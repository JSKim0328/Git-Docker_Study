FORM
FORM 이란?

웹 페이지에서 사용자에게 입력을 받을 때 쓰는 제출 형식. 받은 입력을 action 속성 을 통해서 서버로 보낼 수 있다. method 속성은 데이터 전송방식을 결정하는데, 그 방식은 다음과 같다.
  • GET
    주소에 데이터를 추가하여 전달하는 방식. 데이터가 주소창에 그대로 나타나기에 중요하지 않고 크기가 작은 데이터를 보낼 때 주로 사용된다.
  • POST
    데이터를 따로 첨부하여 전달하는 방식. 크기가 큰 데이터를 더욱 안전하게 보낼 수 있다.

GET 방식: <form action="expample.org/ex.html" method="get"></form>

POST 방식: <form action="expample.org/ex.html" method="post"></form>

input
input 이란?

사용자에게 입력을 받을 수 있는 요소이다. type 속성을 통해 여러 형태로 입력을 받을 수 있다. 타입은 다음과 같은 것들이 같다.
  • text

    텍스트를 입력받을 수 있는 칸이 생긴다.


  • password

    text 와 유사하나 그 내용이 감춰져서 보여진다.


  • hidden

    서버에 전송을 해야하지만 사용자에게 알려줄 필요도, 입력받을 필요도 없는 값을 전송할 때 쓰인다.


  • radio

    여러 개의 옵션 중 하나를 선택할 수 있게 하는 입력 형식이다. 이 때, 옵션들의 name 은 서로 같아야 한다. checked 속성을 통해 기본 선택값을 선택할 수 있다.

  • select

    드랍 다운 리스트의 형태로 여러 개의 옵션 중 하나를 선택할 수 있는 요소이다. select 태그로 감싼 뒤에, option 태그로 옵션들을 나열한다. 이 때, multiple 속성을 사용하면 여러 옵션을 고를 수도 있다. selected 속성으로 초기 선택값을 설정할 수 있다.


  • file

    파일을 업로드할 수 있는 타입이다. 파일 타입의 인풋이 하나라도 있으면 form의 method는 무조건 post가 되어야 한다. enctype 이라는 속성은 인코딩 방식을 지정하는데, 파일이나 이미지를 전송하려면 'multipart/form-data'로 설정한다.


  • button

    버튼 요소는 사용자가 누를 수 있는 버튼을 나타낸다. onclick 속성 등을 통해 버튼을 눌렀을 때의 동작을 정의할 수 있다.


  • submit

    submit 타입은 form의 action에 명시된 서버에 입력값들을 제출하는 버튼이다.
